## Reflection

This assignment was my first experience building a full-stack application using Node.js, TypeScript, and React. Initially, I started with only backend development, but later extended the project to include a frontend dashboard and complete feature flow.

### Key Learnings

- Understood how backend servers work using Express and how APIs are designed
- Learned the request-response cycle and how frontend communicates with backend
- Gained practical experience with TypeScript in both backend and frontend
- Learned how React state (useState, useEffect) controls UI updates
- Built features like baseline selection, compliance balance (CB), banking, pooling, and history tracking
- Understood how application state flows from backend → frontend → UI

### How My Thinking Evolved

At the beginning, I was mostly focused on making things work. I was not very clear about how different parts (frontend, backend, APIs) connect.

As I progressed:
- I started understanding where logic should be written (backend vs frontend)
- I began thinking in terms of **features and user actions**, not just code
- I suggested adding a **history/activity section**, which shows I started thinking from a product perspective
- I moved from basic UI to a more structured dashboard design

This shift from just coding → understanding system flow was an important learning.

### AI Usage

AI helped significantly in:
- Setting up the project (backend + frontend)
- Debugging errors quickly
- Explaining concepts step by step
- Helping structure the UI and features

However, I also learned:
- AI outputs are not always directly usable and need verification
- Real understanding comes from asking follow-up questions and trying things myself
- Debugging issues myself was important for learning

### Challenges Faced

- Setting up Git, npm, and project structure
- Fixing TypeScript and Tailwind CSS configuration issues
- Understanding where to implement features (frontend vs backend)
- Running frontend and backend together and handling API connections
- Debugging environment and dependency issues

### Improvements for Future

- Plan application architecture more clearly before starting
- Improve frontend design and user experience further
- Add data visualization (charts) for better insights

### Conclusion

This assignment helped me understand full-stack development from scratch, including backend logic, API design, frontend integration, and UI development. It also taught me how to effectively use AI as a learning tool rather than just for generating code.

Overall, I gained confidence in building a complete working application and developed a better understanding of how real-world systems are structured.
